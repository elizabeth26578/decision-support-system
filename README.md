# Система підтримки прийняття рішень
## Тема
**Вибір ноутбука для студента-програміста**

## 1. Постановка задачі
Потрібно обрати найкращий ноутбук із кількох альтернатив на основі набору критеріїв.  
Задача є **напівструктурованою**, тому що:
- частина параметрів є числовими і формалізованими;
- остаточне рішення залежить від ваг критеріїв;
- одна альтернатива може бути кращою за одними критеріями і гіршою за іншими.

Тому для задачі доцільно використати **систему підтримки прийняття рішень (СППР)**.

## 2. Формалізація задачі
### Множина альтернатив A
A = {A1, A2, A3, A4}, де:
- A1 — Lenovo IdeaPad Slim 5
- A2 — ASUS Vivobook 15
- A3 — Acer Swift 3
- A4 — HP Pavilion 14

### Множина критеріїв C
C = {C1, C2, C3, C4, C5}, де:
- C1 — Ціна (minimize)
- C2 — Продуктивність (maximize)
- C3 — Автономність (maximize)
- C4 — Вага (minimize)
- C5 — Гарантія (maximize)

## 4. Архітектура системи
Проєкт реалізовано за схемою:

- **Controller** — взаємодія з користувачем через консольне меню.
- **Service** — аналітичний блок: розрахунок рейтингу, пояснення, візуалізація.
- **Data** — збереження та завантаження даних із JSON-файлу.
- **Models** — моделі критеріїв та альтернатив.

Структура директорій:
```text
app/
  controllers/
  services/
  data/
  models/
data/
main.py
```

## 5. Реалізований функціонал
Система дозволяє:
- зберігати предметну область;
- додавати критерії з типами `maximize` / `minimize`;
- додавати альтернативи;
- зберігати матрицю оцінювання;
- обчислювати рейтинг альтернатив;
- визначати найкращу альтернативу;
- пояснювати результат;
- будувати графік рейтингу.

## 6. Аналітичний блок
### Основний метод: Weighted Sum Model
Етапи:
1. Нормалізація оцінок.
2. Урахування типу критерію (`maximize` або `minimize`).
3. Множення нормалізованих значень на ваги критеріїв.
4. Обчислення інтегральної оцінки альтернативи.
5. Сортування альтернатив за спаданням оцінки.

### Бонусний метод: TOPSIS
У проєкті додатково реалізовано метод TOPSIS, що дає змогу порівняти результат іншою стратегією згортки.

## 7. Пояснення рішення
Система виводить:
- найкращу альтернативу;
- внесок кожного критерію;
- тип критерію;
- початкове значення;
- вагу критерію.

Це дає базове пояснення, чому саме така альтернатива стала найкращою.

## 8. Інструкція запуску
### Де це робити
Рекомендовано запускати у:
- **VS Code**
- або **PyCharm**
- або звичайному **терміналі**

```bash
pip install -r requirements.txt
```
3. Запустити програму:
```bash
python main.py
```

## 9. Що показувати на захисті
1. Структуру проєкту (Controller / Service / Data / Models).
2. JSON-файл з критеріями та альтернативами.
3. Додавання нового критерію або альтернативи.
4. Побудову рейтингу методом Weighted Sum.
5. Побудову рейтингу методом TOPSIS.
6. Пояснення вибору найкращої альтернативи.
7. Графік результатів.

## 10. Висновок
Було розроблено програмний продукт класу СППР для вибору ноутбука.  
Система працює з альтернативами, критеріями, матрицею оцінювання, реалізує аналітичний блок, формує рейтинг і пояснює вибір.  
Проєкт придатний для реальної задачі та може бути легко розширений для інших предметних областей.
